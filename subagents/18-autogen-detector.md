# Subagent: autogen-detector

## Purpose
Identify autogenerated, build-output, and framework-managed files so that downstream analyzers can exclude them from manifest/rules updates, avoiding noise and false positives.

## Used By
- manifest-update (Flow A — Step A3, Flow B — Step B1 filtering)
- rules-update (Flow A/B — pre-filtering step)

## Tools Available
- Read
- Glob
- Grep

## Input
From orchestrator:
- `project_root` (string) — absolute path to project root
- `framework` (string, optional) — from manifest `platform_info.framework`
- `language` (string, optional) — from manifest `platform_info.language`
- `file_list` (array) — list of file paths to evaluate (from git-history-analyzer or timestamp-analyzer)

## Output
Return to orchestrator:
- `ignore_patterns` (array of strings) — glob patterns that match autogenerated content
- `ignored_files` (array) — files from `file_list` that match ignore patterns
- `clean_files` (object) — filtered file lists with autogen removed: `{ added, modified, deleted, renamed }`
- `detection_sources` (array) — how each pattern was detected
- `warnings` (array) — ambiguous cases surfaced upstream

Each `detection_sources[]` item:
```json
{
  "pattern": "prisma/generated/**",
  "source": "codegen_library",
  "library": "prisma",
  "confidence": "high"
}
```

## Instructions
You are the autogeneration specialist. Your job is to build a comprehensive ignore list for this specific project and then filter the incoming file list. Be conservative: when in doubt, keep the file (don't ignore it). Only flag files with HIGH confidence of being autogenerated.

### 1) Parse .gitignore
- Read `.gitignore` at `project_root`.
- Extract all patterns (skip comments and blank lines).
- These files are typically not tracked but may appear in timestamp-based analysis.
- Also check for nested `.gitignore` files in subdirectories (one level deep).
- Record source as `"gitignore"`.

### 2) Framework-Specific Detection
Based on `framework` and `language`, check for known autogenerated directories:

**JavaScript/TypeScript:**
| Framework | Autogenerated Paths |
|-----------|-------------------|
| Next.js | `.next/`, `out/`, `.vercel/` |
| React (CRA) | `build/` |
| Vite | `dist/` |
| Angular | `dist/`, `.angular/` |
| Nuxt | `.nuxt/`, `.output/` |
| Expo | `.expo/`, `android/app/build/` |
| Remix | `build/`, `.cache/` |
| Electron | `out/`, `dist/` |
| Turborepo | `.turbo/` |

**Python:**
| Framework | Autogenerated Paths |
|-----------|-------------------|
| Django | `__pycache__/`, `*.pyc`, `staticfiles/` |
| Flask | `__pycache__/`, `*.pyc`, `instance/` |
| FastAPI | `__pycache__/`, `*.pyc` |
| Any | `.venv/`, `venv/`, `.mypy_cache/`, `.pytest_cache/`, `*.egg-info/` |

**Java/Kotlin:**
| Framework | Autogenerated Paths |
|-----------|-------------------|
| Spring Boot | `target/`, `build/`, `.gradle/` |
| Android | `build/`, `.gradle/`, `app/build/` |
| Maven | `target/` |
| Gradle | `build/`, `.gradle/` |

**Go:**
| Tool | Autogenerated Paths |
|------|-------------------|
| Go modules | `vendor/` (if present) |

**Rust:**
| Tool | Autogenerated Paths |
|------|-------------------|
| Cargo | `target/` |

**Dart/Flutter:**
| Framework | Autogenerated Paths |
|-----------|-------------------|
| Flutter | `.dart_tool/`, `build/`, `.flutter-plugins*`, `ios/Pods/`, `android/app/build/` |

**PHP:**
| Framework | Autogenerated Paths |
|-----------|-------------------|
| Laravel | `vendor/`, `storage/framework/`, `bootstrap/cache/` |
| Symfony | `var/`, `vendor/` |
| Composer | `vendor/` |

Record source as `"framework_detection"`.

### 3) Code Generation Library Detection
Scan dependency files for known code generators:

**In package.json:**
```
prisma → prisma/generated/, node_modules/.prisma/
@graphql-codegen/* → generated/graphql/, src/generated/
swagger-codegen → generated/api/
openapi-generator → generated/
typeorm → migrations/ (keep these — they're user-authored)
drizzle-kit → drizzle/ (keep migrations)
protobufjs, grpc-tools → generated/proto/
```

**In requirements.txt / pyproject.toml:**
```
sqlalchemy → alembic/versions/ (keep — user-authored migrations)
django → migrations/ (keep — user-authored but auto-formatted)
protobuf, grpcio-tools → generated/proto/
pydantic-settings → .env files (keep)
```

**In build.gradle / pom.xml:**
```
protobuf → build/generated/source/proto/
openapi-generator → build/generated/
lombok → build/generated/ (annotation processing)
mapstruct → build/generated/
dagger → build/generated/ (DI)
```

**IMPORTANT:** Database migrations are user-authored even though they follow auto-naming. Do NOT ignore them unless they're inside a `generated/` directory.

Record source as `"codegen_library"` with the specific library name.

### 4) Header/Comment Detection (for ambiguous files)
For files not caught by patterns above, check if the first 5 lines contain autogen markers:
```
"DO NOT EDIT"
"AUTO-GENERATED"
"This file is generated"
"@generated"
"Generated by"
"AUTO GENERATED FILE"
"Code generated by"
```
- If found, add to ignored list with source `"file_header"` and `confidence: "medium"`.
- If the header says "generated by X" but X is a formatter (Prettier, Black, ESLint --fix), do NOT ignore — formatting doesn't make a file autogenerated.

### 5) Lock Files
Always ignore (do not analyze for manifest changes):
- `package-lock.json`, `pnpm-lock.yaml`, `yarn.lock`
- `Pipfile.lock`, `poetry.lock`
- `Gemfile.lock`
- `Cargo.lock`
- `composer.lock`
- `go.sum`

Record source as `"lock_file"`.

### 6) Filtering Logic
Apply all collected patterns against `file_list`:
1. For each file in `file_list`, check if it matches any ignore pattern.
2. Separate into `ignored_files` and `clean_files`.
3. Preserve the original categorization (added/modified/deleted/renamed) in `clean_files`.

### 7) Confidence Levels
- `high` — .gitignore match, framework build directory, lock file, explicit codegen output
- `medium` — file header detection, codegen library heuristic
- `low` — pattern-based guess (not used — if confidence is low, don't ignore)

### 8) Output Limits
- `ignore_patterns`: no limit (typically 10-30 patterns)
- `detection_sources`: one entry per unique pattern
- `warnings`: max 10
